# Farm Valley — world dressing & layout

Split out of [player-and-interaction.md](player-and-interaction.md) on 2026-07-09.
The *generation* pipeline lives in [world-generation.md](world-generation.md); this page is
what gets **placed** on the generated world, plus the archipelago's tile geometry.

## Big workshop buildings — multi-tile bottom-anchored scenery (2026-06-05)

The blacksmith and carpenter islands each get a large **2×3-tile (32×48px) building** behind the work-yard: `structure/forge-house` (stone+timber smithy, brick chimney, slate roof, glowing window) and `structure/carpenter-workshop` (green-roofed timber lumber-mill). These are the first non-16×16 sprites: [atlas-recipes](../../games/farm/atlas-recipes/src/) `PixelRecipe` gained optional `width`/`height` (default `size`), and the [atlas-builder](../../tools/atlas-builder/src/index.ts) packs/rasterizes by `recipeWidth/recipeHeight` (atlas widened so the widest frame fits a shelf). They are **static**, so they bake into the static layer via `BIG_STRUCTURES` in [render-systems.ts](../../games/farm/sim-core/src/render-systems/) `iterStaticSprites` (layer 5 — above floor/walls, below fences/props/NPCs). `drawSprite` is center-anchored, so a building of height `hPx` standing on base tile `(tx,ty)` is emitted at center `y = ty*TILE + TILE − hPx/2` (bottom edge on the tile's bottom) and `x = baseTileX*TILE + wPx/2`. **Placement constraint:** each craft island has a vertical through-road *spine* (blacksmith x60–61, carpentry x24–25) that bridges land on top and bottom; the buildings + yard props must keep that spine clear or the island — and anything routed through it (e.g. forest-north + farm-cora via carpentry) — gets walled off. So the forge-house sits on the island's **east** half (x63–64) and the workshop on the **west** half (x21–22). A new `placeFootprint()` in [region-setup.ts](../../games/farm/sim-core/src/world/region-setup/) spawns invisible `solid` blockers over each building's footprint; `placeProps` now makes every prop `solid` by default. The forge also gets an animated **chimney smoke** overlay (`structure/forge-smoke-a/b/c`, cycled in [main.ts](../../games/farm/client/src/main/) like the forge fire). New 16×16 yard detail props: grindstone, coal-pile, ingot-rack (forge); lumber-rack, sawpit, shavings-pile (carpenter).

## Island edges — region-themed margin on every ocean-facing tile (2026-06-05)

Every island is ringed by an edge band on the tiles where its region land meets the ocean, oriented to face the water, and **themed by region**. [`computeWalls()`](../../games/farm/sim-core/src/render-systems/) mirrors `computeShores`: it scans every **region** tile (`regionAt !== null`) and, for each of the 4 sides whose neighbour is non-walkable (ocean / off-grid), emits an edge band rotated to face that side (above −Y → 0, right +X → 90°, below +Y → 180°, left −X → 270°). Bands are authored top-edge-up in [atlas-recipes](../../games/farm/atlas-recipes/src/), drawn at **layer 4** (above ocean=0 / shore foam=1 / plot dirt=2 / bridges=3, below fences=20 and entities). Because it only covers region tiles — never road tiles — the road-only **bridges** (`regionAt === null`) keep their mouths open, so an edge never seals off a crossing.

The material per region is chosen by `edgeFrame(region)`:

| Region | Edge frame | Reads as |
|---|---|---|
| `farm-*` | `tile/shore-sand` | soft sandy beach |
| `fishing-isle`, `fishing-isle-2` | `tile/shore-sand` | sandy beach (they're sand islands) |
| `carpentry` | `tile/wall-wood` | built wooden plank bulwark |
| `blacksmith`, `quarry-*`, village, forests, mill, wells, grove, ice-pond, … | `tile/wall` | hard stone retaining wall |

This **replaced the old wooden-fence texture** that used to ring every island margin: in the archipelago every farm perimeter faces ocean, so `computeFences` previously drew `tile/fence-h` on the entire margin (the "weird fence texture on the island edge"). `computeFences` now only emits a fence where a farm abuts **another land region** (`regionAt(neighbour) !== null` — none in the current layout), and the ocean-facing margins are edges instead. Render-only change: walkability/collision is untouched — Pip/AI still can't step onto ocean tiles (`PlayerControlSystem.canStand` → `isWalkable`, already enforced). `npm run atlas` now bakes 161 frames (`tile/wall`, `tile/wall-wood`, `tile/shore-sand`); the [render-systems test](../../games/farm/sim-core/src/render-systems.test.ts) asserts all three materials are baked and `tile/fence-h` is not.

## Coral zones — semi-transparent deep-water seabed (2026-06-05)

The ocean between islands is dressed with scattered **coral zones** — quiet seabed patches that read as resting **deep below the water surface** (where fish shelter), not bright reefs poking up. They're purely decorative — coral tiles are non-walkable ocean, so they never affect movement or pathfinding (same posture as the bubble fishing-spots).

- **One unified texture, not per-tile stamps (autotiled).** A zone is a CONNECTED patch of many tiles that blend into a single seabed mass, built like the island wall/shore autotiles. Three frames in [atlas-recipes](../../games/farm/atlas-recipes/src/): `tile/coral-fill` (full-bleed interior — solid muted-blue edges so neighbouring fills join seamlessly), `tile/coral-edge` (seabed that fades to water along the top edge), `tile/coral-corner` (fades a corner quadrant). `computeCoral` picks per cell by its 4 coral-neighbours: 4 → fill, one open side → edge rotated to face the water, two adjacent open sides → corner rotated to the open corner (same up=0/right=90/down=180/left=270 rotation convention as `computeWalls`/`computeShores`). Adjacent fills meet with no seam, so a cluster reads as one big reef.
- **"Deep / submerged" look = muted palette + low alpha.** Blue-shifted/desaturated — built from deep ocean (`V`), structure-blue (`S`/`s`) and muted stone (`Q`), with a lone dull-gold (`o`) polyp accent (no vivid reds/oranges). Baked **semi-transparent** at `CORAL_ALPHA` (≈0.4, in [render-systems.ts](../../games/farm/sim-core/src/render-systems/)) so the flowing water pattern shows through and the shapes look murky/submerged.
- **`computeCoral()`** ([render-systems.ts](../../games/farm/sim-core/src/render-systems/)) selects **open-water** candidates only — an ocean tile (non-walkable) with NO walkable neighbour in its full 8-ring — so zones stay out in the sea, clear of the shore foam (`computeShores`) and island edge bands (`computeWalls`). It grows **8 compact, near-circular clusters** (10–17 tiles each — nearest-to-seed frontier growth so blobs fill in round, with real interiors that become fill tiles) via a fixed-seed LCG (deliberately not `Math.random`, so the baked layout is deterministic and stable). The result (~100 tiles, ~⅓ solid fill) is baked once into `iterStaticSprites` at **layer 2** (above the animated water + foam at 0, below bridges=3 / shore / walls).
- **Render-only.** Coral lives only in the static render layer, never in sim state → determinism untouched, no headless re-verify. The [render-systems test](../../games/farm/sim-core/src/render-systems.test.ts) asserts `tile/coral-fill` + `tile/coral-edge` are baked into the static layer.

## Bridges — styled island connectors

Road-only tiles (`regionAt(x,y) === null`, walkable) that span water render as a plank bridge (`tile/bridge-h`, authored horizontal, rotated 90° for vertical spans) instead of `tile/path`. [`computeBridges()`](../../games/farm/sim-core/src/render-systems/) finds them in two passes: (1) road tiles directly touching ocean; (2) a fixpoint fill of road tiles flanked on an axis by ocean-or-deck, which decks the interior of 2-wide spans. A bridge tile's backdrop is `tile/ocean`; `BRIDGE_SET` lets `backdropFrame` suppress the dirt path on those tiles. The `tile/bridge-h` art is a **rope-railed wooden deck** (twisted-rope rails on the long edges — left/right once rotated — support beams, and depth-shaded plank slats). Bridges are **no longer baked**: [`pushBridgeSprites`](../../games/farm/sim-core/src/render-systems/occluders.ts) pushes them each frame at layer 3 with a slow shared-phase **lateral sway** so a whole span swings together without tearing between planks (2026-06-11). The sway axis follows `BridgeTile.runsVertical` (derived from **deck extent** in `computeBridges`, NOT `rotation` — which is 0 for 2-wide vertical spans): vertical decks sway left–right (x), horizontal decks sway up–down (y).

## Craft-NPC idle pose (blacksmith/carpenter)

`WorkNpc` entities ([systems/work-npc.ts](../../games/farm/sim-core/src/systems/cognition/work-npc.ts)) carry an `idlePose` (e.g. `npc/blacksmith/idle`) used while walking between stations **and** at any station whose `pose` is null (e.g. the oven). Previously such stations set `poseFrame = null`, and the snapshot builder's `frame = poseFrame ?? s.frame` fell back to the `structure/blacksmith` *building* sprite — so the smith visibly "became the building" at the oven. The fix is to always assign a person frame.

## Plot layout

Farms lay out plots in a grid via `PLOT_OFFSETS` in [world/region-setup.ts](../../games/farm/sim-core/src/world/region-setup/) (currently `[-2, 1]` → a 2×2 grid spaced ≥2 empty cells apart). Pip starts standing on its first plot tile (`center + PLOT_OFFSETS[0]` on both axes), not the bare farm center — gated on `farmer.player`. Fences (`computeFences` in render-systems.ts) sit on farm *perimeters* only, never on plot tiles.

## Floor tiles & world dressing (2026-06-04)

- **Carpentry floor is now stone.** `backdropFrame` ([render-systems.ts](../../games/farm/sim-core/src/render-systems/)) maps `carpentry` → `tile/carpentry-floor` (a new offset-brick stone-slab tile in [atlas-recipes](../../games/farm/atlas-recipes/src/)), replacing the old dark `tile/wood-plank`. Distinct from the speckled `tile/stone-floor` used by mill/wells so the workshop reads as worked stone.
- **More decoration props.** New `decoration/*` sprites — `barrel`, `crate`, `potted-plant`, `lamp-post`, `signpost`, `hay-bale`, `bush`, `log-stack` — plus `fish/{minnow,bass,salmon}` and `tool/fishing-rod`. All EDG32, 16×16. A purely-decorative `placeProps` batch in [region-setup.ts](../../games/farm/sim-core/src/world/region-setup/) scatters them across the village hub, craft yards, resource zones, and mill yard. **Props are layer-40 sprites only — they do NOT affect walkability/pathfinding** (only `tileFeature` trees/stones block), so placement is visual-only. Rebuild with `npm run atlas` (atlas now has 157 frames).

## RADIAL archipelago layout (240×240) — services centered, farms on the rim

As of 2026-06-09 the world is a **radial archipelago** (was 88×80 grid-band); the map was **160×160** then **grown to 240×240 on 2026-06-12** (uniform position-only scale, SCALE=1.5, center 80→120). Every zone is its own island ringed by ocean, connected **only** by 2-tile-wide bridges (`ROADS` that span water — no land ever touches between islands). The structure is "town + services in the middle, homesteads on the frontier":

- **Scaling model:** island `bounds` are hand-authored at the original 160 scale and run through `scaleB` — center moves ×SCALE from the design origin (80,80), island **size is preserved**, so inter-island gaps open ×1.5 but islands stay the same size. On-island content (décor, NPC stations, footprints, dock tiles) is locked to its island via `scaleAroundNearestIsland` so it rides with the island instead of drifting off. Live tile coordinates therefore come from [world/regions.ts](../../games/farm/sim-core/src/world/regions.ts) at runtime — don't read the 160-scale literals as final positions.
- **Central cluster** — all ~20 functional/decorative islands packed around the map center. **Village is the hub at dead-center (115–126 × 115–126, center 120,120)**; craft islands flank it E/W; resource zones on the diagonals; seasonal zones north; shrine + waterfall on the north-center band (shrine authored +2x to keep its bridge to the village after scaling); fishing isles + harbor + camp south; heritage islets W/E/SW.
- **Two concentric farm rings** ring the cluster near the edges. **Inner ring** (n=9, **R=78**, φ=−90°): the 5 named farms (Pip/Atticus/Hannah/Otto/Cora, 12×12) on the even slots, procedural farm-0..3 (10×10) on the odd slots. **Outer ring** (n=12, **R=108**, φ=−75°): procedural farm-4..15 (10×10). Radii are the original 52/72 ×SCALE. The ring formula (`ringSlotBounds`) + a fixed-seed ±1 organic jitter (fork `farm-ring-jitter`) is in regions.ts; farm-farm and cluster-farm ocean gaps scaled with the world (≥2 holds everywhere).
- **Ranch islands (2026-06-12):** each of the 21 farms has a neighbouring `ranch-N` island (8×8, kind `'ranch'`, theme `'ranch'`) hosting that farm's livestock pens — the farmer crosses a short bridge to tend (tend is gated on being on the ranch). Placed by a deterministic per-farm cardinal-direction search (outward-preferred, falling back to sideways/inward — pure radial-outward is infeasible since the 240 world has no outer margin); each keeps ≥2 ocean gap + a clean farm↔ranch bridge. `ranchForFarm(farmId)` maps farm→ranch.
- **Bridge tree** rooted at the village. `CLUSTER_BRIDGES` (≈20 hand-authored pairs) wires the central islands; `generateFarmSpokes()` hangs each ring farm off the nearest island that yields a clean straight 2-wide bridge through the moat. A `straightBridge()` helper scans every 2-wide window along the islands' overlap and returns the first rect that overlaps no island and edge-touches only its two endpoints — so spokes never clip a third island (e.g. it dodges well-north between quarry-north and heritage-ruin). 41 bridges total; full BFS connectivity from the village center.

The renderer needed **no structural change**: `backdropFrame` paints non-walkable tiles as ocean, `computeShores` foams land/ocean borders, and `computeBridges` decks any road-only tile touching ocean — the whole archipelago is driven from the region bounds + `ROADS`.

**Per-island floor variety** ([static-layer.ts](../../games/farm/sim-core/src/render-systems/) `backdropFrame`): farms/forests/waterfall/camp → seasonal grass; village → dirt + market-floor square; blacksmith → forge-floor; carpentry → carpentry-floor; quarries → quarry-floor; mill/wells → stone-floor; mushroom-grove → mushroom-floor; ice-pond → ice-floor; fishing isles → sand; **shrine → shrine-floor; harbor → dock-floor; heritage-* → heritage-floor** (the last three added with the radial reorg).

**Themed scenery** — each non-farm island carries hand-scattered themed `decoration/*` props (shrine lanterns + torii, harbor buoy/fish-basket/anchor, forest mushrooms/ferns, quarry ore-carts/rubble, mill grain/flour sacks, ice-pond/waterfall cattails, heritage cairns). Props are layer-40 (resource-zone ones are `solid:false` so they never block tree/stone spawns); a `DECORATION_LABELS` entry makes each hover-able. The open-water coral zones + set-piece scatter still auto-derive from the world dims.
