## [2026-06-13] feature | Per-agent BDI jitter — same-kind farmers diverge

Sim goal [randomize-agent-bdi](todos/2026-06-12-randomize-agent-bdi.md). Spike + proposal scope: bake a small per-agent jitter on three scalar BDI knobs ONCE at spawn so same-`kind` farmers (the 16 procedural farms shared the kind default; the 5 named shared hand-tuned constants) no longer behave identically. Decisions (grilled): own rng fork per agent (isolation); moderate spread ±15–20%.

- **New `agents/bdi-jitter.ts`** — `bakeBdiJitter(spec, seed)` returns `{minGoldReserve, riskTolerance, beanValueFactor}`. Each agent's rng is `createRng(seed).fork("bdi:"+name)` — derived **solely from `(seed, name)`**, so adding/removing/reordering a farmer never shifts another agent's draws and **no tick-time RNG stream is perturbed** (the fork advances only a throwaway base). Named farmers' values are the CENTRE of their jitter → character preserved.
- **Three SCALAR knobs** (no queue reordering — same decision *structure*, shifted *thresholds*): `minGoldReserve` ±30% around the spec base; `riskTolerance` continuous [0,1] ±0.15 around a per-kind base (conservative 0.0 / hoarder 0.5 / opportunist 0.7 / aggressive 1.0; augments the 3-level `riskProfile`, drives harbor-contract speculation); `beanValueFactor` golden-bean bid scale ±0.1 around a per-kind base (0.45/0.9/0.7/0.95).
- **Seam.** `setupFarmer(world, spec, seed)` now takes the seed and writes all three onto `desires.data` (open `Record<string,unknown>` bag — no component type change). Read-sites prefer the baked value over the kind literal: `bean-valuation.ts deliberateBean` reads `desires.data.beanValueFactor`; each personality's `deliberateHarborContract` call reads `desires.data.riskTolerance`. Conservative keeps its day-10 relaxation as `Math.max(0.5, baked)` (floor over the baked value).
- **Verification:** typecheck clean all workspaces; full sim-core suite green (756, +5 new `bdi-jitter.test.ts`: determinism, order-independence, same-kind divergence, valid ranges, base-centring mean≈base). Real-run divergence + 3-day/3-seed determinism diff **skipped per user** (constrained hardware) — the unit tests prove the bake is deterministic + order-independent and the per-agent fork guarantees no stream perturbation. ## [2026-06-12] fix | WebGPU renderer review — 5 bugs (instance-buffer clobber, weather alpha offset, water scroll sign, shadows-over-sprites, tint alpha)

Full review of the `webgpu-migration` branch (waves 0–4) against the Canvas2D reference. Five real bugs found and fixed:

1. **Sprite instance-buffer clobbering (critical).** `SpriteBatch.flush()` wrote the SAME GPU buffer at offset 0 once per atlas group (plus per x-ray ghost) while the frame was being encoded. `queue.writeBuffer()` executes on the queue timeline BEFORE the frame's `submit()`, so the last write won for EVERY draw in the pass — with 5+ atlas sheets interleaved by y-sort, nearly all dynamic sprites rendered the wrong instance data. Fix: frame-packing protocol (`begin() → add()× → upload() once → drawRange()` with `firstInstance` selecting each group's range) in [sprite-batch.ts](../packages/engine/src/render/webgpu/sprite-batch.ts); the [renderer](../packages/engine/src/render/webgpu/renderer.ts) packs the whole frame (groups + ghosts) before encoding. Rule of thumb now documented in the file header: **one writeBuffer per buffer per frame, always before encoding draws that read it.**
2. **Weather invisible.** WGSL `struct { color: vec3<f32>, curtain_alpha: f32 }` packs the f32 at byte offset 12 (inside the vec3's tail padding); [weather-pass.ts](../packages/engine/src/render/webgpu/weather-pass.ts) wrote it at offset 16, so the shader read 0 → rain/snow drew fully transparent.
3. **Water scrolled backwards.** Canvas2D's `pattern.setTransform(translate(+scroll))` shows texel `p − scroll` at world point `p`; [water.wgsl](../packages/engine/src/render/webgpu/shaders/water.wgsl) sampled `p + scroll`. Sign flipped (base + swell passes).
4. **Shadows composited above sprites.** Shadow ellipses were drawn on the 2D overlay, which sits on top of the GPU canvas — farmers' drop shadows darkened the farmers (plus an ad-hoc ×0.7 alpha fudge). New [shadow-batch.ts](../packages/engine/src/render/webgpu/shadow-batch.ts): instanced SDF ellipses drawn in-pass between static layer and sprites (Canvas2D order). Premultiplied source-over of black at alpha `a` ≡ canvas `multiply` of black at alpha `a` (both → `dst × (1−a)`), so the look is identical with no fudge factor.
5. **Tint alpha deviation.** Canvas2D drops the `tintRgba` alpha byte (`draw.ts` uses `tint >>> 8`); the GPU path multiplied it into sprite alpha, so e.g. dying-crop tints (`tintFrom(EDG.slate, 0.6)`) rendered at 0.36 instead of 0.6. GPU now drops it too.

Also hardened: `addAtlas`/`bakeStaticLayer`/`bakeWaterPattern` no-op their GPU uploads after device loss (a destroyed device made `copyExternalImageToTexture` throw out of the sim WebSocket handler).

**Verified:** typecheck clean; engine 96 + farm-valley 140 green; headed Chromium via WSLg (`--enable-unsafe-webgpu`) runs the real game on `backend: webgpu` with zero validation errors/device loss, and time-matched screenshot crops vs `--disable-features=WebGPU` (canvas2d) show matching world render (water pattern + speckles, static layer, bridges, buildings, volcano, waterfall mist). Caveat: **headless** Chromium + SwiftShader loses the WebGPU device ~2 frames after first present (clear-only repro confirms — environment artifact, not app code); use headed WSLg Chrome for GPU verification.

Read [The Book of Shaders](https://thebookofshaders.com/) (ch. 5–13: shaping functions, colors, shapes/SDFs, matrices, patterns, random, noise, cellular noise, fBm) and filtered every technique against the actual WebGPU renderer state (`webgpu-migration` branch, waves 0–4). Headline candidates: GPU day/night wash pass (replaces the overlay-2d wash, unblocks compositing), hash/noise UV-warp to de-tile the water, Voronoi caustics on the baked shallow-water band, fBm cloud shadows, and weather/particle parity fixes (round snow, streak taper, polar 8-point star).
